I'm following the Linux Installation and I'm asked to choose a name and a computer's name. What's the difference between a name and a computer's name? How will this information appear in the terminal?

Great question! This is where you give your system its identity. These two names serve different purposes in the "social life" of your computer.

### The Difference

* **Your Name:** This is your **Real Name** or "Full Name" (e.g., *Alex Smith*). It’s mostly cosmetic. It appears on the login screen and in the "Users" settings.
* **Username:** (Which usually follows "Your Name") This is the **short, lowercase name** the system uses internally (e.g., *alex*). You’ll use this to log in.
* **Your Computer's Name (Hostname):** This is the name your laptop uses to identify itself on a **network** (e.g., *legion-laptop*). If you connect to Wi-Fi, other devices or your router will see this name.

---

### How it appears in the Terminal

In Linux, the terminal prompt follows a very specific "Who @ Where" format by default. It looks like this:

`username@hostname:~$`

**For example:**
If your username is **alex** and your computer's name is **legion-y520**, your terminal prompt will look like this:

`alex@legion-y520:~$`



### Why does this matter?

1.  **Context:** The terminal shows you exactly **who** you are acting as (important if you ever switch to the "root" admin user) and **which** machine you are controlling (important if you ever remotely connect to another computer).
2.  **Organization:** The `~` symbol represents your "Home" folder, and the `$` means you are a standard user (not the system administrator).

### Pro-Tips for Choosing Names:
* **Computer Name:** Keep it short and avoid spaces. Use dashes (like `my-laptop`) instead.
* **Username:** Stick to lowercase letters and no spaces. It makes typing commands much faster!

Welcome to the world of Linux "Permissions"! You haven't done anything wrong, but you’ve run into a classic security feature.

Because you mounted that drive at the "Root" level (the `/` folder), the system thinks that folder belongs to the **System Administrator (root)**, not to **your user account**. That little padlock icon is the computer’s way of saying, "You can look, but you can't touch."

Here is the breakdown of what's happening:

### 1. What is the `lost+found` folder?
Don't worry, you didn't put it there. Every Linux drive formatted as **Ext4** creates this folder automatically. 
* **Its purpose:** If your computer ever crashes or loses power and the file system gets slightly corrupted, Linux runs a check when it restarts. If it finds "homeless" pieces of data that it can't identify, it stuffs them into `lost+found` so you can try to recover them.
* **Can you delete it?** You shouldn't. Just ignore it; it’s like a digital "junk drawer" for emergencies.

### 2. Why is there a padlock icon?
In Linux, folders have "Owners." Right now, the owner of `/data` is the system. Since your user account doesn't own it, you don't have permission to create new folders or drag images into it yet.

### 3. How to fix the shortcut and permissions
We need to tell Linux that **you** are the boss of that folder. Open your terminal (press `Ctrl + Alt + T`) and type the following command carefully:

```bash
sudo chown -R username:username /data
```
*(Replace `username` with the actual username you chose during installation. For example, if your name is alex, type `alex:alex`).*

**What this does:**
* `sudo`: Runs the command as an administrator.
* `chown`: Short for "Change Owner."
* `-R`: Operates "Recursively" (applies to everything inside the folder too).
* `/data`: The target folder.

After you run that and enter your password (you won't see dots as you type, just hit Enter), **the padlock icon should disappear.**

### 4. Did you create the shortcut correctly?
Yes! `Ctrl + Shift + Drag` is the correct way to create a "Symbolic Link" (shortcut) in Linux Mint. 

Once the padlock is gone, you can open that `/data` folder on your desktop, right-click, and create a new folder called "Ebooks" or "Images."
